1. Codekwaliteit & Architectuur (Clean Code)
[ ] Naamgeving: Variabelen en functies hebben duidelijke, beschrijvende en logische namen.
camelCase voor functies, functienamen en variabelen in het Nederlands.
Namespaces gebruiken hoofdletters. 
[ ] Structuur: De afgesproken mappenstructuur (voor onderhoudbaarheid/schaalbaarheid) is strikt gerespecteerd. Logica staat in de juiste lagen/bestanden.
[ ] DRY (Don't Repeat Yourself): Er is geen onnodige dubbele code geschreven.
[ ] SQL Conventies: Alle SQL-queries zijn correct en consequent hoofdlettergevoelig geschreven (bijv. SELECT * FROM table).
[ ] Opkuis: Alle debug-code (zoals console.log(), var_dump(), print_r(), of hardcoded testdata) is verwijderd.
[ ] Functieouput typen: meegeven wat de functie returned.
2. Testen & Functionaliteit
[ ] Acceptatiecriteria: Aan alle specifieke eisen van de User Story is voldaan.
[ ] Documentatie: Feature is volledig gedocumenteerd. (input, output, gebruik, etc) in de code
[ ] Kritieke Flows: Belangrijke processen (zoals checkout, registratie, inloggen, wishlist toevoegen) zijn uitgebreid getest en werken feilloos.
[ ] Geen Dode Links: Alle (nieuwe) links, knoppen en formulieren op de pagina werken correct.
[ ] Geen Regressie: De nieuwe code breekt geen bestaande functionaliteit op de website.
[ ] HTML / CSS validatie: HTML / CSS moet zonder problemen valideren.
3. UX, UI & Responsiveness
[ ] Responsiveness: De feature / pagina is getest en werkt goed op zowel mobiel als desktop.
[ ] Loading times/SEO: SEO en laadsnelheid meten. 
[ ] Cross-browser: De website werkt en toont correct in de afgesproken webbrowsers (bijv. Chrome, Firefox, Safari, Edge).
[ ] Navigatie & Zichtbaarheid: De navigatiestructuur is duidelijk. De belangrijkste acties (aankoop, registratie, inloggen, wishlist, contact) springen direct in het oog.
4. Veiligheid & Database (Aanvullingen)
[ ] Security: Formulieren en database-inputs zijn beveiligd tegen veelvoorkomende aanvallen (zoals SQL-injecties via prepared statements en XSS).
[ ] GDPR / Privacy: Er is geen gevoelige data (zoals wachtwoorden) onversleuteld opgeslagen of gelogd.
5. Proces & Oplevering
[ ] Versiebeheer: De code is op de juiste branch gepusht en voorzien van heldere commit messages. (Aanvulling)
[ ] Peer Review: Merge requests altijd samen behandelen
[ ] Demo / Showcase: De feature is kort gedemonstreerd aan de Product Owner ter goedkeuring.
